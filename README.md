# Google Drive File Manager

Проект для автоматизации работы с файлами на Google Диске через API v3. Скрипт автоматически создает структуру папок, загружает файлы (обновляя их, если они уже существуют, вместо создания дубликатов), получает прямые ссылки на просмотр и скачивает файлы обратно в локальную папку.

## Как установить

### 1. Настройка Google Cloud
Для работы скрипта необходимы права доступа к API:
1. Зайдите в [Google Cloud Console](https://console.cloud.google.com/).
2. Создайте проект и включите **Google Drive API**.
3. Создайте учетные данные (**OAuth 2.0 Client ID**).
4. Скачайте JSON-файл с ключами.
5. **Переименуйте файл в `credentials.json`** и поместите его в папку с проектом (рядом с `main.py`).

### 2. Установка библиотек
Python3 должен быть уже установлен. 
Затем используйте `pip` (или `pip3`, есть конфликт с Python2) для установки зависимостей:
```bash
pip install -r requirements.txt
```
*(Примечание: основные библиотеки — `google-api-python-client` и `google-auth-oauthlib`)*

## Как Запустить проект

Вы можете запускать скрипт как с параметрами по умолчанию, так и указывая свои настройки через консоль.

### Вариант 1: Обычный запуск
Используются дефолтные значения (папка "Projects", подпапка "Python", картинка "utka.png"):
```bash
python main.py
```
При первом запуске откроется окно браузера для авторизации в Google-аккаунте.

### Вариант 2: Запуск с аргументами
Можно переопределить названия папок, типы файлов и ID родителя:
```bash
python main.py --folder_name "MyBackups" --subfolder_name "Images" --mime_type "image/jpeg"
```

**Список доступных аргументов:**
* `--parent_id`: ID папки, в которой нужно создать новую папку.
* `--folder_name`: Имя корневой папки.
* `--subfolder_name`: Имя вложенной папки.
* `--mime_type`: Тип загружаемого файла.
